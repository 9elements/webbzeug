General

Introduction

.webbzeug is a web-based tool that allows you to create procedural textures by stacking and rendering actions. The idea is based on farbrausch's werkkzeug tool. Actions are seperated into three groups: Generative, Processive and Memory actions. This manual explains all actions and how they should be used.

The source code is avaiable at Github.com.

Generative Actions

Cell

The cell action generates cell textures based on the Voronoi diagram using the given cell size, type and seed. There are two types of cells: Balls and Mosaic. The generated texture is tileable. All inputs are overwritten. View source on Github.com

Circle

The circle action draws a circle in the given color with the given size and position. The circle is drawn on the first input, if given. View source on Github.com

Flat

The flat action fills the image with the given color. All inputs are overwritten. View source on Github.com

Fractal

The fractal action fills the image with a fractal noise using the given seed and roughness. All inputs are overwritten. View source on Github.com

GlowRect

The GlowRect action draws a rectangle with a glowing border using the given position, size and radius. The rectangle is drawn on the first input, if given. View source on Github.com

Pixels

The pixels action fills the image with random grayscale pixels based on the given seed. All inputs are overwritten. View source on Github.com

Rectangle

The rectangle action draws a rectangle in the given color with the given size and position. The rectangle is drawn on the first input, if given. View source on Github.com

Processive Actions

Blur

The blur action blurs the input using the given strength and type. The blur action needs exactly one input. View source on Github.com

Combine

The combine action combines two inputs using the given blend mode. Available blend modes are Darken, Lighten, Multiply, Add, Substract, Divide. The combine action needs exactly two inputs. View source on Github.com

HSCB

The HSCB action changes the hue, saturation, contrast and brightness of the first input. It needs exactly one input. View source on Github.com

Invert

The invert action inverts all colors of the input. It needs exactly one input. View source on Github.com

Light

The light action uses two inputs to generate lighting. The first input is the image the light is applied on, the second input is the normal map. The light action needs exactly two inputs. View source on Github.com

Mask

The mask action uses the first input as a mask to combine the second and third input. The first input should be a grayscale image. The mask action needs exactly three inputs. View source on Github.com

Mirror

The mirror action mirrors the image vertically or horizontically. It needs exactly one input. View source on Github.com

Move

The move action moves the input by the given x and y offset. It needs exactly one input. View source on Github.com

Normal

The normal action creates a normal map for the given input. It needs exactly one input. View source on Github.com

Repeat

The repeat action repeats the input using the given scrolling offset and blending method. It needs exactly one input. View source on Github.com

Rotozoom

The rotozoom action rotates and zooms the input using the given parameters. It needs exactly one input. View source on Github.com

Distort

This action distorts the map in 1st input, depending on the content of the second input. The blue channel regulates the distortion for the x direction, the green for the y direction. View source on Github.com

Memory Actinos

Load

The load action loads the input with the given ID. It needs exactly one input. View source on Github.com

Save

The save action saves the input with the given ID. It can later be loaded using the load action. It needs exactly one input. View source on Github.com
